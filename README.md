## Test Suite Specifications

Generated by `tests_generator.cpp` using a fixed RNG seed (`12345`, re-seeded deterministically per `(id, n)` pair so re-runs always produce identical files).

### Sizes

| Suite | Count | N (elements) | Output directory |
|-------|------:|-------------:|-----------------|
| Small | 20 | 20,000 | `./small/` |
| Big | 20 | 10,000,000 | `./big/` |

File naming: `small_team6_<id>.in` / `big_team6_<id>.in` (e.g. `small_team6_01.in`).

### File format

```
<N>
<v1> <v2> ... <vN>
```

First line is the element count; second line is space-separated integers.

### Individual tests

| # | Name | Pattern | Value range | Designed to stress |
|---|------|---------|-------------|-------------------|
| 1 | All zeros | Every element is `0` | `{0}` | Algorithms with poor equal-key handling; best case for many sorts |
| 2 | All equal (non-zero) | Every element is `42` | `{42}` | Same as above but with a non-zero sentinel, confirms no special-zero short-circuit |
| 3 | Sorted ascending | `0, 1, 2, …, N-1` | `[0, N-1]` | Already-sorted best case; worst case for naive Quicksort and Bubble/Insertion/Gnome |
| 4 | Sorted descending | `N-1, N-2, …, 0` | `[0, N-1]` | Reverse-sorted worst case for Bubble, Insertion, Gnome; worst case for naive Quicksort |
| 5 | Random uniform | Uniform random integers | `[0, INT_MAX]` | Average-case baseline across the full 31-bit positive range |
| 6 | Tiny range | Uniform random integers | `[0, 100]` | High duplicate density; stresses equal-key partitioning in Quicksort |
| 7 | Small range | Uniform random integers | `[0, 1000]` | Moderate duplicates; intermediate between tests 6 and 5 |
| 8 | Binary | Uniform random integers | `{0, 1}` | Extreme duplicate density; worst case for comparison-based sorts with poor pivot selection |
| 9 | Nearly sorted | Sorted ascending with `⌈N/1000⌉` random swaps | `[0, N-1]` | Algorithms that exploit pre-sortedness (Insertion, Gnome, TimSort-like); nearly-best-case |
| 10 | Few unique (5 values) | 5 random values from full range, randomly distributed | `[0, INT_MAX]` (5 distinct) | Quicksort equal-key worst case while keeping Radix working across all bits |
| 11 | Pipe organ (`/\`) | Ascending first half, descending second half | `[0, N-1]` | Ideal for Patience Sort (2 runs); exposes naive Quicksort median-of-3 weaknesses |
| 12 | V-shape (`\/`) | Descending first half, ascending second half | `[0, N/2-1]` | Mirror of pipe organ; tests algorithms that rely on monotone-run detection |
| 13 | Sawtooth | Repeating ramp `0…period-1`, period = `⌈N/100⌉` | `[0, period-1]` | Many short ascending runs; stresses merge-based run detection and Patience Sort |
| 14 | Two sorted halves | `[0…N/2-1]` concatenated with `[0…N/2-1]` | `[0, N/2-1]` | Ideal single-merge step for Mergesort; exposes algorithms that don't exploit existing order |
| 15 | Median killer | Interleaved `1,2,3,…` and `N,N-1,N-2,…` | `[1, N]` | McIlroy adversarial input — forces worst-case O(N²) on median-of-3 Quicksort |
| 16 | Integer boundaries | Mix of `0`, `1`, `INT_MAX-1`, `INT_MAX`, `INT_MAX/2`, `INT_MAX/2+1` | 6 boundary values | Overflow-prone pivot arithmetic in Quicksort; extreme bit patterns for Radix |
| 17 | All large | Uniform random integers in upper half of range | `[INT_MAX/2, INT_MAX]` | Forces Radix to process all high-order bits; large key comparisons for comparison sorts |
| 18 | One out of place | Sorted ascending with `INT_MAX` placed at index 0 | `[0, N-1] ∪ {INT_MAX}` | Single misplaced maximum — one wrong element that must travel the full array length |
| 19 | Sorted with outliers | Sorted ascending with ~1% of positions replaced by random values | `[0, INT_MAX]` | Nearly-sorted with noise; adaptive sorts (Insertion, Gnome) remain fast, others unchanged |
| 20 | Small positives | Uniform random integers | `[0, 65536]` | Best case for Radix Sort (only 17 bits needed); baseline for comparison against other sorts |
